## Step 3: PQ（Product Quantization 乘积量化）编码与写入索引文件

### 3.1 PQ 训练

**参数**：8 维向量，子空间数=2，每子空间 4 维，码字数=256（8 bit）。

**PQ 码本是全局训练的**（不是每个分区独立训练）：

1. 从全量数据中采样训练数据
2. 对 L2/Cosine 距离：计算采样向量与其最近质心的**残差**（`vector - centroid`）
3. 在残差向量上训练**一个全局 PQ 码本**
4. 所有分区共用同一个码本

## 查询示例

查询向量 `[1.0, 2.0, 3.0, 4.0, 0.1, 0.2, 0.3, 0.4]`，`nprobes=2`，`k=3`。

### 1. 计算到质心距离

- 到质心 0: 0.12
- 到质心 1: 12.8
- 到质心 2: 35.2

### 2. 选 nprobes=2 分区

分区 0, 1

### 3. 在分区 0 内 PQ 搜索

**关键：查询向量减去质心，得到查询残差**

查询向量: [1.0, 2.0, 3.0, 4.0, 0.1, 0.2, 0.3, 0.4]
质心 0:    [1.05, 2.03, 3.09, 3.95, 0.15, 0.25, 0.35, 0.38]

查询残差 = 查询向量 - 质心 0
= [-0.05, -0.03, -0.09, 0.05, -0.05, -0.05, -0.05, 0.02]

**用查询残差查 codebook 距离表**：

1. 查询残差拆分子空间：

   - 子空间 0: [-0.05, -0.03, -0.09, 0.05]
   - 子空间 1: [-0.05, -0.05, -0.05, 0.02]
2. 预计算 lookup table：

   - 子空间 0: 查询残差到 256 个码字的距离
     lookup_table[0] = [d(q0, code_0), d(q0, code_1), ..., d(q0, code_255)]
   - 子空间 1: 同样
     lookup_table[1] = [d(q1, code_0), d(q1, code_1), ..., d(q1, code_255)]
3. 对分区 0 内每条向量的 PQ 码，查表累加：
   向量 A 编码: [119, 201]
   近似距离 = lookup_table[0][119] + lookup_table[1][201]

**查码本距离表**，快速计算与分区 0 内 3,333 条向量的近似距离：

具体如何查距离表：

```text
┌────────┬────────┬────────┬────────┬────────┐
│        │ 码字 0  │ 码字 1 │ 码字 2  │ 码字255 │
├────────┼────────┼────────┼────────┼────────┤
│ 码字 0  │ 0.0    │ 0.15   │ 0.30   │ 0.25   │
├────────┼────────┼────────┼────────┼────────┤
│ 码字 1  │ 0.15   │ 0.0    │ 0.35   │ 0.20   │
├────────┼────────┼────────┼────────┼────────┤
│ 码字 2  │ 0.30   │ 0.35   │ 0.0    │ 0.18   │
├────────┼────────┼────────┼────────┼────────┤
│ 码字255 │ 0.25   │ 0.20   │ 0.18   │ 0.0    │
└────────┴────────┴────────┴────────┴────────┘
```

**示例：查询 `[119, 201]` 与向量 `[3, 5]` 的距离计算**

```
查询编码: [119, 201]
向量编码: [3, 5]

近似距离 = lookup_table[119][3] + lookup_table[201][5]
         = 0.20          + 0.15
         = 0.35
```

**对比原始 L2**：

- PQ 方式：查 2 次表，加 1 次 = **2 次内存访问 + 1 次加法**
- 原始 L2：算 8 维差值，平方，累加，开方 = **大量浮点运算**

对 3,333 条向量：

- PQ：3,333 × 2 次查表 + 3,333 次加法
- 原始：3,333 × 8 次减法 + 平方 + 累加

**两阶段搜索**：

**阶段 1：PQ 粗排（快速近似）**

- 用 lookup table 查所有 3,333 条的 PQ 码
- 得到近似距离，排序取 Top (nprobes * k * refine_factor) 条
  如 k=10, refine_factor=5 → 取 Top 50 条

**阶段 2：精确重排（可选）**

- 对这 50 条，用原始向量（不是 PQ 码）算精确 L2 距离
- 重新排序，取最终 Top 10

**refine_factor 作用**：

- =1：PQ 粗排后取 Top k，不重排（最快，精度最低）
- =5：PQ 粗排后取 Top 5k，对这 5k 精确重排（平衡）
- =None/0：跳过 PQ，直接精确计算（最慢，精度最高）

### 4. 在分区 1 内同样搜索

查询向量 - 质心 1 = 查询残差，同样编码后查距离表（[3, 2]）。

然后去和所有3,333 条都这样做，得到每条向量L2距离，找到topK最近的向量.

### 5. 合并取 Top-3

**多分区结果合并**：

分区 0 PQ 粗排 → Top 50 条（含近似距离）
分区 1 PQ 粗排 → Top 50 条（含近似距离）

合并 100 条 → 按近似距离排序 → 取 Top 15 条

如果启用了 refine：
对这 15 条加载原始向量 → 算精确距离 → 重排 → 取 Top 3

返回最终 Row ID 和精确距离。

返回最近邻的 Row ID 和距离。
