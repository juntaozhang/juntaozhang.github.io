# Kernel Term

## DMA（Direct Memory Access）

定义：硬件设备可以直接访问内存，无需 CPU 参与
```
例子：
传统文件拷贝（CPU 参与）：
CPU 读取磁盘 → CPU 写入内存 → CPU 读取内存 → CPU 写入磁盘
CPU 像搬运工，一条一条搬运数据

DMA 方式：
CPU 告诉 DMA 控制器：从 A 地址搬运 4KB 到 B 地址
DMA 硬件自动搬运，CPU 可以做其他事情
完成后 DMA 中断通知 CPU
```

## CPU copy

定义：CPU 执行 memcpy 指令，逐字节拷贝数据

```
CPU copy 的代价：
for (int i = 0; i < 4096; i++) {
dest[i] = src[i];  // 每次循环：load + store 指令
}

4096 字节 = 4096 × 8 比特
假设 CPU 3GHz，每个内存操作 4 个周期
= 4096 × 4 / 3,000,000,000 ≈ 5 微秒
```

## 用户态/内核态切换
CPU 特权级的改变，保护系统安全

切换开销：\
用户态 → 内核态：
1. 保存用户寄存器到内核栈：~100 纳秒
2. 加载内核栈指针：~50 纳秒
3. 改变特权级（CPL）：~20 纳秒
4. 内核检查参数：~100 纳秒
   小计：~270 纳秒

内核态 → 用户态：
1. 恢复用户寄存器：~100 纳秒
2. 恢复用户栈：~50 纳秒
3. 改变特权级：~20 纳秒
   小计：~170 纳秒

总切换开销：~440 纳秒

## Scatter-Gather DMA
网卡可以从多个不连续的内存地址读写数据

传统方式：\
[协议头 54B][数据 4096B]  ← 必须连续内存

需要 CPU 拼接：copy 到一个连续 buffer

Scatter-Gather DMA：\
网卡可以同时发送：
- 段 1：0x9000_0000（协议头 54B）
- 段 2：0x7000_0000（数据 4096B）
  在物理网络上自动合并成一个帧

优点：
- 无需 CPU 拼接
- 数据可以留在原处（用户态内存）
