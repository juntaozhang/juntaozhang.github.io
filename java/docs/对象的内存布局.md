# Java 对象的内存布局

## 对象头（Header）

| 字段 | 大小 | 说明 |
|------|------|------|
| **Mark Word** | 8 字节（64 位 JVM） | 存储哈希码、GC 信息、锁信息（轻量/偏向锁） |
| **Class Pointer** | 4 或 8 字节 | 指向方法区中类的元数据（即 class 对象） |

## 普通对象内存布局

```
┌──────────────────┐
│ 对象头（Header）   │  ①
├──────────────────┤
│ 实例数据（Fields） │ ②
├──────────────────┤
│ 对齐填充（Padding）│ ③
└──────────────────┘
```

## 数组对象内存布局

```
┌────────────┐
│ 对象头      │   ← 一般 12 字节（64 位 + 压缩指针）
├────────────┤
│ 数组长度    │   ← 4 字节（int 类型）
├────────────┤
│ 数组数据[]  │   ← 元素按顺序存储
├────────────┤
│ Padding    │   ← 补齐为 8 字节倍数
└────────────┘
```

- 数组是连续内存块：访问速度快，CPU 缓存友好
- 数组元素是定长：JVM 可以直接通过 `base + index * size` 访问
- 越界检查：JVM 在数组访问时自动做 bounds check（除非 JIT 优化掉）
- 数组在 GC 中标记为"普通对象"但有 `length` 字段

### 数组大小计算示例

| 数组类型 | 每个元素占用 | 示例（3 个元素）总大小计算 |
|----------|------------|------------------------|
| `int[]` | 4 字节 | 12（头）+ 4（length）+ 3×4 = 12 + padding = 32 |
| `long[]` | 8 字节 | 12 + 4 + 3×8 = 40 → padding 到 40，无需补齐 |
| `byte[]` | 1 字节 | 12 + 4 + 3×1 = 19 → 补齐到 24 |
| `Object[]` | 4 字节（压缩指针） | 12 + 4 + 3×4 = 28 → 补齐到 32 |

## `Object[]` 中元素实际对象大小不同，Java 是怎么处理的？

```java
Object[] arr = new Object[3];
arr[0] = new String("hello");
arr[1] = new Integer(123);
arr[2] = new int[]{1, 2, 3};
```

| 数组位置 | 存的内容 | 实际对象存放在哪里 |
|---------|---------|------------------|
| `arr[0]` | 地址引用 A | 堆上的 `String("hello")` |
| `arr[1]` | 地址引用 B | 堆上的 `Integer(123)` |
| `arr[2]` | 地址引用 C | 堆上的 `int[] {1, 2, 3}` |

`arr[]` 里存的只是地址（指针/引用），不管这个对象是 10 字节还是 100 字节，都不影响数组中的槽大小！
