# Java 类加载机制

类加载机制是指 JVM 将类的字节码文件（`.class`）动态加载到内存，并在运行时将其转化为 `Class` 对象的过程。

## 加载过程

### 加载（Loading）

- 类加载器读取 `.class` 文件的二进制数据（可能来自文件系统、网络、JAR 包等）
- 转换为 `Class` 对象
- 并放入方法区（MetaSpace）

### 连接（Linking）

| 阶段 | 说明 |
|------|------|
| **验证（Verification）** | 校验 class 文件格式、字节码合法性、安全性等 |
| **准备（Preparation）** | 为类的静态变量分配内存，并设置默认值（非初始化值） |
| **解析（Resolution）** | 将符号引用（如类名、字段名）转为直接引用（内存地址） |

#### 符号引用 vs 直接引用
在java中，一个java类将会编译成一个class文件。在编译时，java类并不知道引用类的实际内存地址，因此只能使用符号引用来代替。

比如org.simple.People类引用org.simple.Tool类，在编译时People类并不知道Tool类的实际内存地址，
因此只能使用符号org.simple.Tool(假设)来表示Tool类的地址。而在类装载器装载People类时，
此时可以通过虚拟机获取Tool类的实际内存地址，因此便可以将符号org.simple.Tool替换为Tool类的实际内存地址，即直接引用地址。

### 初始化（Initialization）

执行类中定义的 `static` 初始化块和静态变量的初始化语句。

## 类加载器

| 类加载器 | 说明 |
|----------|------|
| **Bootstrap ClassLoader** | 加载 JDK 的核心类（rt.jar、java.base 等） |
| **Extension ClassLoader** | 加载 JDK 扩展类（jre/lib/ext/） |
| **Application ClassLoader** | 加载用户类路径中的类（classpath） |
| **自定义 ClassLoader** | 可通过继承 `ClassLoader` 实现加载加密类、网络类等 |

## 双亲委派模型

每个类加载器在加载类时，首先将请求委托给父加载器，只有当父加载器无法加载时，才尝试自己加载。

**优点：**

- 防止类的重复加载
- 保证核心类安全（如不会加载一个伪造的 `java.lang.String` 类）

## 案例

### ChildFirstURLClassLoader

Spark 为避免与系统类冲突，使用**子优先加载策略**。

### ContextClassLoader

有些框架（如 JDBC 驱动、SPI 插件机制）需要加载"用户代码"中的类，但自己是在"系统类加载器"或更高层中创建的，无法看到用户类路径下的内容。

```java
Class<?> userClass = Thread.currentThread().getContextClassLoader()
                          .loadClass("com.mycompany.MyUDF");
```

相比起 `Class.forName()` 依赖当前类的类加载器，这种方式是显式指定"使用线程上下文的加载器"，从而"反向"加载用户类。
