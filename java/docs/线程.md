# Java 线程

## 进程、线程、协程

| 概念 | 说明 |
|------|------|
| **进程** | 操作系统资源分配的最小单位，拥有独立内存空间，进程间隔离 |
| **线程** | CPU 调度的最小单位，属于进程内部，共享内存 |
| **协程（虚拟线程）** | 用户态的轻量线程（协作式调度），切换开销更小，非抢占式 |

Java 21+ 可使用 `startVirtualThread` 创建虚拟线程。

### Kotlin 协程

Kotlin 协程的本质是"用状态机实现了一个用户态的协程调度系统"，一切阻塞/挂起点都变成状态编号，`resume` 恢复执行靠 `Continuation`。它是一种编译器+运行时框架层面的用户态任务调度模型。

### Java 虚拟线程

Java 虚拟线程的本质就是**函数栈帧的挂起和恢复**，只不过这个"栈帧切换"是在用户态完成的，而不是依赖操作系统内核线程上下文切换，因此代价更小。

虚拟线程从一个 platform thread 切换到另一个，并不会触发传统意义上的"OS 线程上下文切换"。这个切换完全由 JVM 在用户态完成，代价极低。它不是线程之间的切换，而是"运行时绑定线程"的重新安排。

操作系统线程上下文切换之所以代价高，是因为它涉及内核调度器、寄存器保存、栈切换、缓存失效等硬件和系统层面资源操作；而用户态切换仅发生在用户空间，是纯粹的内存级状态切换，速度快一个数量级甚至多个数量级。

## 线程有哪些状态？如何转换？

Java 线程状态：

| 状态 | 说明 |
|------|------|
| **NEW** | 新建状态，线程被创建但未启动 |
| **RUNNABLE** | 可运行状态，包括运行中（Running）和就绪（Ready） |
| **BLOCKED** | 阻塞状态，等待监视器锁 |
| **WAITING** | 等待状态，无限期等待另一个线程的特定操作 |
| **TIMED_WAITING** | 超时等待，在指定时间内等待 |
| **TERMINATED** | 终止状态，线程执行完毕 |

状态转换：

```
NEW → start() → RUNNABLE → 获取CPU → Running
                                    ↓
                          等待锁 → BLOCKED → 获取锁 → RUNNABLE
                                    ↓
                          wait()/join() → WAITING → notify()/被唤醒 → RUNNABLE
                                    ↓
                          sleep(time)/wait(time) → TIMED_WAITING → 时间到 → RUNNABLE
                                    ↓
                               run()结束/异常 → TERMINATED
```

## 特权级切换（用户态 → 内核态）

### Ring 3 → Ring 0

线程上下文切换：

- 前者（用户态 → 内核态）只涉及到部分寄存器
- 后者（线程上下文切换）涉及到所有寄存器
- 后者涉及到调度子系统，因为需要调度其他线程使用 CPU
