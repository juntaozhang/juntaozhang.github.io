
## Master/Slave

### Ctrl panel
如何横向扩展
- 普通 hash 分片
  - 优点：简单
  - 缺点：扩容破坏旧规则
- [一致性哈希](https://bytebytego.com/courses/system-design-interview/design-consistent-hashing): 固定数量逻辑分片 + 动态映射物理机
  - 扩容只需要迁移部分元数据
    - 新机器在迁移完成之前，可以自动forward到源节点
    ```text
    hash 0~511  → Node A
    hash 512~1023 → node B
    
    新增一个节点
    hash 0~255  → Node A
    hash 256~511  → Node new [if not exist forward to Node A]
    hash 512~1023 → node B
    ```
- 类似redis 固定16384个槽位，每个分片对应一些slot，扩容只需要迁移部分slot数据
- 每个对象元信息包含
  - version：[objectid，offset size，nodeid], checksum(校验和，防止文件损坏)

## Decentralized
- minio
  - gossip：节点直接心跳
  - Erasure Set：每个node 维护这个set
  - 单个小文件入缓冲区，不立即写盘，类似“预写日志”（源码中是小文件和meta放一起了）？？
  - 每个文件都有自己的meta，meta 也写入到磁盘

1GB 文件上传 MinIO 时，先按 128MB 默认阈值拆 8 个分片，每个分片经 hash 路由到不同 8+4 Erasure Set；每个分片拆 8 份数据、算 4 份校验，合并写入对应 ES 的 12 块磁盘；全程无中心节点，任意节点可接收请求，上传后元数据记录分片位置，下载时自动拼接还原，容错 4 块盘故障。
